The `CheckboxGroup` component is a special compoment, designed to group a set of `Checkbox` components together. It is a useful component for grouping related checkboxes and allowing users to select multiple options. You can also choose whether to arrange these checkboxes horizontally or vertically.

## Usage

To use the `CheckboxGroup`, you'll typically need to import both the `CheckboxGroup` and `Checkbox` components from the package:

```jsx
import { CheckboxGroup, Checkbox } from '@syntatis/kubrick';
```

In your WordPress admin application, you can create a group of checkboxes by adding the `label`, `name`, and add multiple `Checkbox` components inside the `CheckboxGroup`. Each checkbox within the group should have a unique value to represent different options.

```jsx
<CheckboxGroup
	label="Hide on screen"
	name="hide_on_screen">
  <Checkbox
    label="Permalink"
    value="permalink"
  />
  <Checkbox
    label="Excerpt"
    value="excerpt"
  />
  <Checkbox
    label="Author"
    value="author"
  />
</CheckboxGroup>
```

Similar to the `Checkbox` component, you can mark the group as disabled by passing the `isDisabled` prop:

```jsx {3} collapse={5-17}
<CheckboxGroup
	label="Hide on screen"
	isDisabled
	>
	<Checkbox
		label="Permalink"
		value="permalink"
	/>
	<Checkbox
		label="Excerpt"
		value="excerpt"
	/>
	<Checkbox
		label="Author"
		value="author"
	/>
/>
```

Or, mark the group as read-only with the `isReadOnly` prop:

```jsx {3} collapse={5-17}
<CheckboxGroup
	label="Hide on screen"
	isReadOnly
	>
	<Checkbox
		label="Permalink"
		value="permalink"
	/>
	<Checkbox
		label="Excerpt"
		value="excerpt"
	/>
	<Checkbox
		label="Author"
		value="author"
	/>
/>
```

Both the `isDisabled` and `isReadOnly` props will apply to all checkboxes within the group.

### Description

You can also add a description to the `CheckboxGroup` component to provide additional information, instruction, or context about the group of checkboxes:

```jsx {3} collapse={5-8}
<CheckboxGroup
	label="Hide on screen"
	description="Select the items you want to hide on the screen."
>
	<Checkbox
		label="Permalink"
		value="permalink"
	/>
	<Checkbox
		label="Excerpt"
		value="excerpt"
	/>
	<Checkbox
		label="Author"
		value="author"
	/>
</CheckboxGroup>
```

## Events

The `CheckboxGroup` component has its own event handler.

| Prop | Description |
| --- | --- |
| `onChange` | Triggered when the check state changes. |

You can listen to the `onChange` event to handle the changes in the group of checkboxes. The event will pass an array of values representing the checked checkboxes:

```jsx
import { useState } from 'react';

function App() {
	const [selection, setSelection] = useState([]);

	return (
		<CheckboxGroup
			label="Hide on screen"
			onChange={setSelection}
		>
			<Checkbox
				label="Permalink"
				value="permalink"
			/>
			<Checkbox
				label="Excerpt"
				value="excerpt"
			/>
			<Checkbox
				label="Author"
				value="author"
			/>
		</CheckboxGroup>
	);
}
```

If the `permalink` and `author` checkboxes are checked, the `selection` state will be `['permalink', 'author']`.

## Styles

The `CheckboxGroup` component comes with a few static selectors, prefixed with `.kubrick-CheckboxGroup-`, that you can use to select and customize the component and its elements.

<div className="table-classes">
| Selector | Description |
| --- | --- |
| `root` | The root element of the checkbox group. |
| `label` | The element that wraps checkbox group label. |
| `items` | The element that wraps the list of `Checkbox` components. |
| `description` | The element that wraps the desceription. |
</div>
